## Project Overview

CampusConnect is a Flutter-based mobile app providing a centralized platform for COMSATS students to manage events, trips, announcements, donations, and registrations — replacing scattered WhatsApp groups and noticeboards.

---

## Tech Stack

| Layer | Technology | Cost |
|-------|-----------|------|
| Frontend | Flutter (Dart) | Free |
| Auth | Firebase Auth | Free tier |
| Database | Cloud Firestore | Free tier |
| Storage | Firebase Storage | Free tier (5GB) |
| Notifications | Firebase Cloud Messaging | Free |
| Maps | flutter_map + OpenStreetMap | Completely Free |
| State | Provider | Free |

## Features

- **Authentication** — Email/Password + Google Sign-In
- **Events** — Browse, filter by category, register/unregister
- **Trips** — Book seats, view itinerary, see destination on map
- **Donations** — Campaign progress bars, quick donation amounts
- **Announcements** — Real-time feed with push notifications
- **Profile** — View registered events/trips, student info
- **Admin Panel** — Add/delete events & trips, post announcements

## Setup Instructions

### 1. Prerequisites

```bash
flutter --version   # Must be 3.x
dart --version      # Must be 3.x
```

Install Flutter: https://flutter.dev/docs/get-started/install

### 2. Clone & Install Dependencies

```bash
flutter pub get
```

### 3. Firebase Setup (Required)

1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Create a new project: `campus-connect-comsats`
3. Enable these services:
   - **Authentication** → Email/Password + Google Sign-In
   - **Cloud Firestore** → Start in test mode
   - **Firebase Storage** → Start in test mode
   - **Cloud Messaging** → Auto-enabled

4. Install FlutterFire CLI:
```bash
dart pub global activate flutterfire_cli
```

5. Configure Firebase:
```bash
flutterfire configure
```
This auto-generates `lib/firebase_options.dart` with your real credentials.

6. Download config files:
   - `google-services.json` → place in `android/app/`
   - `GoogleService-Info.plist` → place in `ios/Runner/`

### 4. Run the App

```bash
# Android (connected device or emulator)
flutter run

# iOS (Mac only)
flutter run -d ios

# Release APK
flutter build apk --release
```

## Firestore Security Rules (Recommended)

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Users can read/write their own data
    match /users/{userId} {
      allow read: if request.auth != null;
      allow write: if request.auth.uid == userId;
    }
    // Events - anyone can read, only admins write
    match /events/{eventId} {
      allow read: if request.auth != null;
      allow write: if get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
    }
    // Trips - same as events
    match /trips/{tripId} {
      allow read: if request.auth != null;
      allow write: if get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
    }
    // Donations - anyone can read, admins create campaigns, all can add records
    match /donations/{donationId} {
      allow read: if request.auth != null;
      allow create: if get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
      allow update: if request.auth != null;
      match /records/{recordId} {
        allow read, write: if request.auth != null;
      }
    }
    // Announcements
    match /announcements/{announcementId} {
      allow read: if request.auth != null;
      allow write: if get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
    }
  }
}
```

## Making a User Admin

After signing up, open Firebase Console → Firestore → `users` collection → find your user document → change `role` field from `"student"` to `"admin"`.
